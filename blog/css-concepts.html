Classes vs IDs

February 7, 2016

What are two attributes that allow designers to uniquely style elements in a webpage?

In HTML and CSS, designers and programmers strive to be as specific as possible when writing their code. Many different semantic tags can be used to designate the header, footer, section, paragraph and many other elements in a webpage. Within CSS, designers have a plethora of options, choosing everything from font style, background color, border design, table margins, text positioning and more.

Two attributes that allow designers to uniquely style elements in a webpage are classes and ids. Classes should be applied when the designer has several elements that require the same styling. This class can be used repeatedly, which sames time and allows for code to be written efficiently. On the opposite end of the spectrum, id's are applied when the designer has only one element that should be styled differently from the rest of the content in the webpage. The specificity of id's ensures that a specific style is adhered to and cannot be applied to other elements.

What are the best practices associated with classes?

When deciding on a whether or not to create a class, the first question a developer should ask is, "Will these style elements be applied to other areas of my website?" Some of the common elements in a particular class could be related to font style, font color, or border design.
